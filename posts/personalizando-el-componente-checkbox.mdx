Como en todos los componentes que creemos, trataremos de generar el HTML mas simple posible para dejar una estructura facil de seguir. En este caso, luego de mucha simplificación, quedaria asi:

```Html
<div class="radio">
  <input class="radio-input" type='radio' name='frontend-army' />
</div>
```

La idea de esto es hacer que el input sea invisible y que el div que lo contiene sea el que cambie entre un estado sin seleccionar y uno seleccionado. Para hacer visible el estado seleccionado entonces, agregaremos un pseudoelemento `::before` asi no andamos ensuciando el HTML con cosas que son esteticas.

Si les parece, vamos yendo paso a paso y vamos viendo como queda todo.

Primero definimos el contenedor. A este lo targetearemos con el selector `.radio` y le ponemos propiedades para definir el tamaño del componente, para darle estilo al fondo del mismo y por ultimo le agregamos `position: relative` ya que queremos que tanto el `input` como el `::before` que usaremos dentro de este contenedor tengan un tamaño y posicion en relacion a este.
```CSS
.radio {
  width: 1.5rem;
  height: 1.5rem;
  position: relative;
  background: #FAFAFA;
  border: 1px solid #DDD;
  border-radius: 100%;
}
```

Ahora sigamos con el input, el cual targetearemos usando el selector `.radio-input` al que le agregaremos estilos que se vuelva invisible, otros para que se vea la manito al pasar el mouse por arriba y por ultimo la propiedad `inset: 0` decimos que se expandirá a todos los limites del primer ancestro con posicion distinta a estatico (es decir el elemento `.radio` en este caso).

React es una biblioteca poderosa para construir interfaces de usuario dinámicas y receptivas. Sin embargo, como cualquier herramienta, también tiene sus propias trampas, y una de ellas es la posibilidad de generar memory leaks cuando se utiliza el hook useEffect de manera incorrecta.

## ¿Qué es un memory leak?

Un memory leak (fuga de memoria) ocurre cuando un programa de computadora asigna memoria de manera continua sin liberarla, lo que puede resultar en una pérdida gradual de rendimiento o incluso en el bloqueo del programa.

## ¿Cómo se generan los memory leaks en React con useEffect?

Olvidar limpiar efectos anteriores: Si un efecto de useEffect realiza suscripciones a eventos o crea temporizadores, es crucial limpiarlos cuando el componente se desmonta. De lo contrario, estas suscripciones y temporizadores seguirán activos, consumiendo memoria incluso después de que el componente ya no esté en uso.

Dependencias incorrectas o faltantes: useEffect depende de un array de dependencias que determina cuándo se debe volver a ejecutar el efecto. Si este array está mal configurado o si algunas dependencias faltan, el efecto puede ejecutarse más veces de lo necesario, lo que puede llevar a fugas de memoria.

Referencias a funciones o variables obsoletas: Si el efecto hace referencia a variables o funciones que ya no son necesarias pero que aún están en el ámbito de cierre, puede provocar que esas referencias no se liberen de la memoria.

## ¿Cómo evitar los memory leaks en React?

Siempre limpia los efectos anteriores utilizando el mecanismo de retorno de useEffect.
Verificá y configurá correctamente las dependencias del useEffect para evitar ejecuciones innecesarias.
Utiliza referencias ref para almacenar variables y funciones que necesiten persistir entre renderizaciones, y asegurate de limpiarlas cuando ya no sean necesarias.
En conclusión, aunque useEffect es una herramienta poderosa para manejar efectos secundarios en React, también requiere un manejo cuidadoso para evitar posibles memory leaks. Siguiendo buenas prácticas y prestando atención a los detalles, podemos garantizar un rendimiento óptimo y una experiencia de usuario sin problemas en nuestras aplicaciones React.
